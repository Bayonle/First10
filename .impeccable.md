# Impeccable Design Context — First10

## Design Context

### Users
FRSC Ogun Command duty officers working the dispatch desk — 24/7 shifts in bright
Nigerian offices (daylight glare, aging monitors). Their job: scan a live queue,
spot the real high-severity incident in seconds, act (dispatch/arrive/transport),
and close the loop. Time-to-action target ≤15s per ticket (paper §7.1). Secondary
surface: the dev-only local-chat cockpit used by the engineering team to simulate
bystanders — functional, not precious.

### Brand Personality
Official, urgent, legible. The console should feel like *operational equipment* —
an incident logbook/dispatch manifest, not a startup dashboard. Calm authority:
urgency comes from information hierarchy (severity, status), never from visual
noise. FRSC's livery is amber-yellow and deep blue; the UI acknowledges it through
warm paper-toned neutrals and a restrained action blue, with signal red reserved
exclusively for severity/danger.

### Aesthetic Direction
**"Dispatch manifest"**: light theme (bright-office glare rules out dark), warm
manila-paper surfaces, ink-dark text, squared "stamp" status chips in uppercase
microtype, tabular/mono figures for times and coordinates. Structure over
decoration: full hairline borders, flat surfaces, no shadows/glass/gradients.
Anti-references: dark ops dashboards with neon accents; generic admin templates;
card grids with icon headers.

Fonts: **Archivo** (headings/stamps — transport-signage grotesque),
**Public Sans** (body/UI — civic-form clarity), **Spline Sans Mono**
(timestamps, coordinates, IDs).

### Design Principles
1. Severity is the loudest thing on screen — red is spent nowhere else.
2. Evidence vs. inference stays visually distinct (reporter messages ≠ system/AI
   annotations) — D-013 is a design rule, not just a data rule.
3. Every state is legible at arm's length in sunlight: high contrast, ≥1.25 type
   scale steps, uppercase stamps for statuses.
4. Density without clutter: the queue is a manifest list, not a card gallery.
5. Motion only for state changes (new ticket arriving, status advancing) — subtle,
   fast, ease-out.
